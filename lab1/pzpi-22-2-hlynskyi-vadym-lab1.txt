1.	Бізнес-вимоги

1.1 Передумови
У сучасних корпоративних середовищах залученість та мотивація співробітників є критично важливими для підвищення продуктивності та утримання кадрів. Традиційні робочі середовища часто не мають достатніх інтерактивних і мотиваційних механізмів, що може призводити до зниження залученості. Система управління роботою з елементами гейміфікації спрямована на вирішення цієї проблеми шляхом впровадження ігрових елементів у повсякденні завдання, винагороди за їх виконання та відстеження прогресу до особистих і організаційних цілей.

1.2 Бізнес-можливості
Впровадження системи відкриває можливості для підвищення залученості, мотивації та продуктивності співробітників. Це також сприяє створенню більш приємної та винагороджувальної атмосфери на робочому місці, заохочуючи командну співпрацю та досягнення особистих цілей.
Ринок систем гейміфікації робочих процесів зростає, і наша система вирізняється інтеграцією з IoT для автоматичного контролю доступу (наприклад, до кавомашини) на основі досягнень користувачів.
Модель монетизації передбачає базовий безкоштовний доступ із платними підписками для розширених функцій, таких як аналітика, кастомізація винагород та інтеграція з IoT. Додатковий дохід можливий через партнерства з виробниками IoT-пристроїв.



2.	Концепція рішення

2.1 Окреслення концепції
Система управління роботою з елементами гейміфікації — це платформа, яка поєднує управління завданнями з ігровими механіками для підвищення мотивації та продуктивності співробітників. Основна ідея полягає в тому, щоб перетворити рутинні робочі процеси на захоплюючий досвід, де користувачі отримують віртуальну валюту за виконання завдань і можуть обмінювати її на привілеї, такі як доступ до IoT-контрольованої кавомашини.

2.2 Головна функціональність
Система підтримує дві основні ролі користувачі (співробітники) та адміністратори.

2.2.1 Функціональність для користувачів
•	MF-1. Реєстрація та авторизація
Користувачі можуть зареєструватися через соціальні мережі (Google, Facebook) або корпоративні акаунти. Після входу вони отримують доступ до персонального кабінету з інформацією про завдання, віртуальну валюту та прогрес.
•	MF-2. Управління завданнями та винагородами
Співробітники бачать список завдань (наприклад, Завершити звіт), виконують їх і отримують віртуальну валюту. Цю валюту можна обміняти на доступ до IoT-пристроїв, як-от кавомашина.
•	MF-3. Мобільний додаток
Дозволяє переглядати завдання, відмічати їх виконання, отримувати сповіщення про нові цілі та відстежувати прогрес у реальному часі.

2.2.2 Функціональність для адміністраторів
•	MF-4. Управління користувачами та ролями
Адміністратори можуть додавати користувачів, призначати їм ролі, встановлювати права доступу та групувати співробітників у команди.
•	MF-5. Панель адміністратора
Інтерактивна панель із графіками продуктивності, статистикою завершення завдань і налаштуваннями системи винагород (наприклад, визначення кількості віртуальної валюти за завдання).
•	MF-6. Контроль доступу через IoT
Адміністратори налаштовують правила доступу (наприклад, 100 одиниць валюти = доступ до кавомашини). IoT-пристрої автоматично активуються на основі досягнень користувачів.


3.	Рамки та обмеження

3.1 Рамки первинного запуску
Перший реліз системи зосередиться на базовій функціональності, необхідній для запуску та тестування концепції.

3.1.1 Серверна частина
•	Зберігання даних про користувачів, завдання, віртуальну валюту та IoT-доступ.
•	Логіка нарахування валюти наприклад, 10 одиниць за просте завдання, 50 за командний проєкт.
•	API для взаємодії з IoT-пристроями та мобільним додатком.
•	Автентифікація через OAuth 2.0.

3.1.2 IoT-клієнт
•	Інтеграція з IoT-пристроями (наприклад, кавомашиною) для контролю доступу.
•	Передача даних у реальному часі коли користувач набирає потрібну кількість валюти, сервер надсилає сигнал IoT-пристрою.
•	Підтримка базового протоколу (наприклад, MQTT).

3.2 Рамки наступних випусків
Наступні релізи розширять можливості системи для зручності користувачів і адміністраторів.

3.2.1 Веб-клієнт
•	Повноцінний веб-інтерфейс на React із підтримкою drag-and-drop для управління завданнями.
•	Розширена аналітика графіки продуктивності, теплові карти активності.

3.2.2 Мобільний клієнт
•	Офлайн-режим для перегляду завдань і синхронізації після підключення.
•	Push-сповіщення про дедлайни та нові винагороди.
•	Підтримка Android (Kotlin) та iOS.
4.	Бізнес-контекст

4.1 Робоче середовище
Система функціонуватиме як
•	Веб-платформа Доступна через браузери (Chrome, Firefox) для десктопних користувачів.
•	Мобільний додаток Для Android (Kotlin) та iOS, з акцентом на зручність у дорозі.
•	IoT-інтеграція Безпечне підключення до корпоративної мережі для взаємодії з пристроями (наприклад, через VPN або шифрований канал).
Технологічний стек
•	Сервер C# (ASP.NET Core) із базою даних PostgreSQL.
•	Веб-клієнт React.js.
•	Мобільний клієнт Kotlin для Android, React Native для iOS.

4.2 Зацікавлені сторони
•	Співробітники Користувачі системи, які виконують завдання та отримують винагороди.
•	Адміністраторименеджери Налаштовують систему та відстежують прогрес.
•	Керівництво компанії Зацікавлені в підвищенні продуктивності та ROI від впровадження.
•	IT-відділ Відповідає за інтеграцію та підтримку системи.
